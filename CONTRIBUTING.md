# Contributing

Thanks for your interest in contributing.

This repository contains small Bash and Python utilities.
The goal is to keep them simple, reliable, and easy to maintain.

---

## Requirements

- Bash >= 3.2
- Python >= 3.12
- Git >= 2.8

Minimum versions may be increased if newer, widely used versions provide clear benefits.

---

## How to Contribute

1. Fork the repository
2. Create a branch
3. Make your changes
4. Test locally
5. Open a pull request

Keep changes focused and small.

---

## Coding Style

- Keep scripts readable
- Avoid unnecessary complexity
- Use clear variable and function names
- Do not add heavy dependencies

---

## Testing

Before submitting, make sure:

- Scripts run without errors
- Existing functionality still works

Include example commands in the PR if relevant.

---

## Commits

Use clear commit messages, for example:

Types:
- feat
- fix
- docs
- refactor
- test
- chore

If a change affects only one script, prefix the type with `bash:` or `py:`.

Format:

- bash:<type>: short description
- py:<type>: short description

Examples:

- fix: handle missing repo directory
- feat: add dry-run option
- bash:fix: improve error handling
- py:feat: add config validation
- docs: update usage example

---

## Pull Requests

Pull requests should explain:

- What changed
- Why it changed
- How it was tested
